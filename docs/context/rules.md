# Agent rules (TLDR)

Sourced from `agents.md`. Full history in repo root `agents.md` (collapsed completed tasks).

## Must

- Show **version number** in feature messages
- Run tests before push; never push crashing builds
- Keep messages **brief** — user is often on-call
- **80%** viewport = transcription + translation
- **20%** = scoreboard / tracking
- Propose changes precisely (what → what)

## Code style

- Simple, clear, token-efficient
- Looks good at **900×600**, scales up
- Refactor/cleanup after satisfied features

## Scoreboard

- Monthly target pacing, daily mins/AR$, breaks (90m budget)
- Labels toggle; hover tooltips when hidden

## Translation

- Poor translation > none; don't destroy translations on bubble split
- Don't burn free API tiers
